# 教材插图性别意识研究 - 项目说明

## 项目目标
分析中小学教材插图中的人物性别与职业分布，研究教材中的性别意识表现。

## 工作流程
1. **初始识别**：多模态大模型识别教材PDF中的人物信息 → 输出Excel/JSON
2. **职业分类**：将识别出的职业描述匹配到《中国职业大典》分类体系 → 一位码(0-10) + 三位码(10100-70400)
3. **统计分析**：汇总多本教材数据，进行性别×职业交叉统计

## API配置
- **初始识别阶段**：OpenRouter API，配置在 `1.初始识别/.env`
- **职业分类阶段**：阿里通义千问API，配置在 `2.职业分类/.env`
- 两个阶段的API密钥分开存储，各自有独立的 .env 文件

## 识别输出字段规范
识别阶段输出两种表格，字段统一为：
| 字段名 | 说明 |
|--------|------|
| identifier | 人物标识（姓名或外貌描述） |
| profession | 职业自然语言描述（如"医生"、"学生"） |
| gender | 性别（男/女/未知） |
| source_type | 判断依据（插图/插图和文本） |
| scenario | 场景类型 |
| page | 页码 |

### 两种统计方式
1. **人头计数表**：每个识别到的人物一条记录（原有逻辑）
2. **独立职业表**：按 (page, profession, gender) 去重，每页每种职业每个性别只计一次（解决集体照权重过高问题）

### 独立职业表的正确派生流程
**重要**：独立职业表必须从已分类的人头计数数据中派生，不能独立分类。原因：
- 独立职业表去重后没有 `identifier` 列，profession="未知"的记录无法走轨道B分类
- 正确流程：识别 → 分类人头计数（有identifier可走轨道A+B）→ 从已分类人头计数按 (page, profession, gender) 去重得到独立职业

### 命名标准化
分析阶段对原始职业文本做标准化合并（如 "老师"→"教师"），在 `PROFESSION_MERGE` 字典中定义。

## 职业分类规范
- 一位码：0-10共11类（含自定义的0=学生/儿童、9=家庭角色、10=非从业人员-其他）
- 三位码：78类（代码10100-70400），非从业人员（学生/儿童/家庭角色）沿用一位码的0/9/10
- 分类采用"双轨制"：已知职业用全量分类表，profession为"未知"的用identifier在限制分类表(0/9/10)中匹配

## 执行顺序
1. 配置 `1.初始识别/.env` 和 `2.职业分类/.env`
2. 运行 `1.初始识别/batch_recognition.py` → 生成识别结果（人头计数+独立职业）
3. 运行 `2.职业分类/batch_classification.py` → 分类人头计数，自动派生独立职业
4. 运行 `3.结果分析/batch_statistics.py` → 生成统计汇总Excel
5. 运行 `3.结果分析/analysis_profession.py` 和 `analysis_visualization.py` → 生成图表

## 图表规范
- **百分比横轴统一尺度**：同一张图中，凡是以百分比为横轴的子图，所有子图必须使用同一个X轴范围，以便跨版本/跨性别直接视觉比较
- 纵轴（类别标签轴）可以不同，但横轴刻度必须一致

## 错误教训（2026.3）

### 独立职业去重逻辑错误
**问题**：初版代码按 `(page, profession)` 去重生成独立职业表，用 `gender=mode()` 赋值性别。这导致：
1. 同一页同一职业的不同性别被合并，少数性别的职业记录被抹去
2. 独立职业丰富度低于人头计数丰富度（不合理）
3. 独立职业表丢失 `identifier` 列，在职业分类阶段无法走轨道B（profession="未知"的记录分类失败）

**修正**：
- 去重键改为 `(page, profession, gender)`，保留每个性别的不同职业
- 分类流程改为：先对人头计数做完整分类（含轨道A+B）→ 再从已分类人头计数派生独立职业
- 分析阶段加入命名标准化（合并"老师"→"教师"等变体）

**核心教训**：
- **与性别相关的信息（核心研究目标）不能随意舍去或简化，修改前必须先询问用户**
- 去重、聚合、mode赋值等操作会改变数据分布，尤其当研究目标就是"分布差异"时更需谨慎
- 数据处理链中上游的错误会级联影响下游所有分析，要特别注意第一步的逻辑正确性
